# Signal

Signal is a career intelligence system designed to help a senior product-minded engineer find exceptional job opportunities without turning job hunting into a second job.

The goal is not to maximize the number of applications.
The goal is to maximize the probability of finding the right company, role, culture, compensation, and product fit.

## Why this exists

Most job platforms are optimized for volume.

They show many openings, many companies, and many alerts.

Signal is optimized for judgment.

It researches companies, understands opportunities, evaluates fit, prepares strategies, and only asks for human attention when there is something worth reviewing.

## Core idea

Signal is not a job board.
Signal is not a resume bot.
Signal is not an application spammer.

Signal is an intelligence layer between the job market and the candidate.

It separates signal from noise.

## Product principles

- Quality over quantity.
- Company first, role second.
- Human approval for every external action.
- Every recommendation must be explainable.
- Every token must generate value.
- Knowledge is an asset.
- The system should protect the user's attention.
- The MVP exists for one purpose: help the user get the right job within six months.

## Current status

This repository is in product discovery.

No application code should be written until the product documentation, domain model, and initial architecture decisions are mature enough to guide implementation.
